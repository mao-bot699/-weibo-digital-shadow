## 1. 准备事项

开始之前，你需要准备：

- **一台电脑**（macOS / Linux / Windows 均可）
- **Python 3.9 或更高版本**
- **一个微博账号**（用于获取 Cookie）
- **一个 LLM API Key**（用于 AI 分析，兼容 OpenAI 格式）

---

## 2. 获取微博 Cookie

Cookie 相当于你的登录凭证，工具通过它代替你访问微博公开数据。

### 操作步骤

1. 打开浏览器（推荐 Chrome 或 Edge），访问 [weibo.com](https://weibo.com) 并登录你的微博账号
2. 登录后，按 **F12** 打开开发者工具
3. 切换到 **Network（网络）** 标签页
4. 按 **F5** 刷新页面
5. 在请求列表中找到任意一个请求，点击它
6. 在右侧找到 **Headers（请求头）** 部分，向下滚动找到 **Cookie** 字段
7. 复制 **整行 Cookie 值**（非常长的一串字符，以 `SCF=` 或 `SUB=` 开头）

### 注意事项

- Cookie 通常 **几周后会过期**，过期后需要重新获取
- 不要把你的 Cookie 分享给任何人，它相当于你的账号密码
- 如果获取时遇到滑块验证，请手动通过后再操作

## 3. 获取 LLM API Key

工具需要调用大语言模型来生成分析报告。支持两种 API 服务：

### 选项 A：StepFun（推荐，国内稳定）

1. 访问 [platform.stepfun.com](https://platform.stepfun.com)
2. 注册账号并实名认证
3. 进入「API Keys」页面，创建一个 Key
4. 复制 Key 备用

### 选项 B：OpenAI

1. 访问 [platform.openai.com](https://platform.openai.com)
2. 注册账号并充值
3. 进入「API Keys」页面，创建一个 Key
4. 复制 Key 备用

## 4. 安装与配置

### 第一步：克隆仓库

```bash
git clone https://github.com/mao-bot699/-weibo-digital-shadow.git
cd -weibo-digital-shadow/数字影子-公开版
```

### 第二步：安装依赖

```bash
pip install flask flask-cors openai requests
```

如果遇到权限问题，可以加 `--user`：

```bash
pip install --user flask flask-cors openai requests
```

### 第三步：配置 Cookie

```bash
# 复制模板文件
cp config.py.example config.py
```

然后用文本编辑器打开 `config.py`：

```python
"""
微博 Cookie 配置
"""

WEIBO_COOKIE = "把你刚才复制的Cookie粘贴到这里"
```

### 第四步：配置 LLM

打开终端，设置环境变量：

**如果用 StepFun：**
```bash
export CLAUDE_API_KEY="你的StepFun API Key"
export CLAUDE_BASE_URL="https://api.stepfun.com/step_plan/v1"
export CLAUDE_MODEL="step-3.7-flash"
```

**如果用 OpenAI：**
```bash
export CLAUDE_API_KEY="你的OpenAI API Key"
export CLAUDE_BASE_URL="https://api.openai.com/v1"
export CLAUDE_MODEL="gpt-4o"
```

> 提示：把上面三行加到 `~/.zshrc` 或 `~/.bashrc` 里，以后就不用每次重新设置了。

## 5. 三种使用方式

### 方式一：Web 页面（推荐新手）

```bash
python3 server.py
```

终端会显示：

```
==================================================
🕵️  数字影子 服务启动中...
   页面地址：http://localhost:8080
==================================================
```

用浏览器打开 `http://localhost:8080`，按页面提示操作即可。

#### Web 页面操作流程

**第 1 步：输入 UID**

在页面顶部的输入框中输入微博 UID。怎么找 UID？

- 打开 [weibo.com](https://weibo.com)
- 进入你想分析的用户的个人主页
- 看地址栏：`weibo.com/u/后面跟着的数字就是UID`，比如 `weibo.com/u/1749127163`
- UID 就是 `1749127163`

**第 2 步：点击「确认」**

页面会显示用户的头像、昵称、粉丝数、微博数。

**第 3 步：调整参数**

拖动滑块设置要获取的微博数量（建议 20-50 条，越多分析越详细，但耗时更长）。

**第 4 步：点击「生成数字影子报告」**

等待约 30-60 秒，AI 会完成分析。

**第 5 步：查看报告**

报告分为多个板块：

| 板块 | 说明 |
|------|------|
| 分析报告 | AI 生成的 11 维度深度画像 |
| 互动关系图谱 | 该用户微博中 @最多的 Top 10 用户 |
| 多平台线索 | 从微博中匹配到的其他平台账号 |
| 追问补充 | 输入问题或补充信息，继续深入分析 |
| 手动补充数据 | 粘贴其他平台的帖子内容，生成跨平台画像 |

**第 6 步：下载报告**

点击「下载报告 (TXT)」按钮保存到本地。

---

### 方式二：命令行（适合批量分析）

```bash
# 基本用法
python3 main.py 1749127163

# 指定获取微博数量
python3 main.py 1749127163 --max-posts 30

# 导出报告到文件
python3 main.py 1749127163 --max-posts 30 --export
```

报告会直接显示在终端，加 `--export` 会同时保存到 `exports/` 目录。

---

### 方式三：Streamlit（交互式网页）

```bash
streamlit run app.py
```

浏览器会自动打开，提供带侧边栏配置的交互界面。

## 6. 功能详解

### 6.1 分析报告

每次分析生成约 2000-4000 字的深度报告，覆盖 11 个维度：

1. **人物总画像** — 身份标签、公开形象 vs 内在性格、人生阶段
2. **性格与心理特征** — 语言风格推断性格、情绪管理、价值观、MBTI 猜测
3. **生活方式与日常规律** — 作息推断、通勤出差、饮食运动、消费偏好
4. **兴趣爱好与关注领域** — 核心兴趣、行业圈子、娱乐偏好
5. **主题演变分析** — 内容主题是否随时间变化、是否存在阶段转换
6. **社交网络特征** — 粉丝互动模式、转发原创比例、影响力层级
7. **互动关系图谱** — 高频 @对象及关系推断、社交圈层特征
8. **多平台线索分析** — 微博中匹配到的其他平台账号及可信度评估
9. **语言与表达特征** — 语言风格、网络流行语、标点 emoji 习惯
10. **数据时间线** — 发博规律、活跃期/沉默期、主题演变趋势
11. **综合总结** — 核心特征、三个关键词、出人意料的发现

### 6.2 互动关系图谱

自动提取用户微博中所有 @提及 的用户，统计出现频率，展示 Top 10 互动对象。

**能看出什么：**
- 互动最多的是谁（可能的关系：同事、朋友、家人、偶像）
- 社交圈层是否集中在某个行业
- 是主动 @ 别人的类型，还是被别人 @ 的类型

### 6.3 多平台线索

从用户微博的简介和正文中，自动匹配以下平台的账号线索：

| 平台 | 匹配方式 |
|------|---------|
| Twitter / X | 匹配 twitter.com/xxx 链接或 "@Twitter: @xxx" 格式 |
| Instagram | 匹配 instagram.com/xxx 链接或 "@Instagram: @xxx" 格式 |
| B站 | 匹配 space.bilibili.com/xxx 链接或 "B站: xxx" 格式 |
| 小红书 | 匹配 "小红书号：xxx" 格式 |
| 抖音 | 匹配 "抖音号：xxx" 格式 |

> 注意：以上线索仅为自动匹配结果，不一定属于该用户本人，仅供参考。

### 6.4 追问补充

在已有报告的基础上，输入补充信息或问题，AI 会基于原有报告继续深入分析。

**示例用法：**
- 「这个用户是科技创业者，补充一些行业背景」
- 「分析一下他近半年的情绪变化」
- 「对比他和普通科技博主的差异」

### 6.5 手动补充数据

如果知道该用户在其他平台的账号，可以复制其他平台的帖子内容粘贴进来，AI 会结合微博数据生成更完整的跨平台画像。

**操作步骤：**
1. 在平台下拉框选择来源（Twitter / B站 / 小红书等）
2. 在文本框粘贴帖子内容（可以多条，空行分隔）
3. 点击「补充分析」

## 7. 常见问题

### Q：找不到 UID 怎么办？

在 [weibo.com](https://weibo.com) 进入用户主页，地址栏 `weibo.com/u/` 后面的数字就是 UID。也可以在页面搜索用户后从搜索结果中获取。

### Q：提示「Cookie 已过期」怎么办？

重新按步骤 2 获取新的 Cookie，更新 `config.py` 即可。

### Q：分析报告被截断了怎么办？

报告被截断通常是因为获取的微博数量太少。建议把滑块调到 30 条以上再试。如果还是截断，说明该用户的微博内容太少，可分析的数据有限。

### Q：搜索功能为什么搜不到人？

搜索功能使用了微博移动端 API，反爬机制较强，有时会返回空结果。建议直接输入 UID 使用，更加稳定。

### Q：分析报告准确吗？

报告基于 AI 对公开数据的分析和推断，结论仅供参考。数据不足的地方会明确标注「基于现有数据无法确定」。AI 的分析不是事实判断，而是一种基于数据的合理推测。

### Q：支持分析自己的微博吗？

支持。用你自己的微博账号登录浏览器，按步骤 2 获取 Cookie，然后输入你自己的 UID 即可。

### Q：需要一直开着电脑吗？

Web 页面模式下需要保持终端运行。命令行模式下分析完成后会自动退出。

### Q：分析一个用户需要多长时间？

取决于获取的微博数量和报告复杂度，通常 30-90 秒。

### Q：API 调用收费吗？

取决于你使用的 LLM 服务：
- **StepFun**：按 token 计费，一次完整分析（50 条微博）大约消耗 8000-15000 tokens，费用约 ¥0.1-0.5
- **OpenAI**：按 token 计费，GPT-4o 一次分析约 $0.05-0.15

### Q：可以分析别人的微博吗？

可以分析任何公开微博账号，工具仅获取公开可见的数据。建议仅用于正当用途，尊重他人隐私。

### Q：生成的报告存在哪里？

- **Web 模式**：点击下载按钮保存到本地
- **命令行 + `--export`**：保存在 `exports/` 目录下
- **追问后**：最新的报告会保存在浏览器的 localStorage 中
